Disease Simulation

School Project

Java  ·   JavaFX
Muli-threading  ·  Concurrency  ·   Liveness  ·  Collaborative Project

Objective:

Create a 2D disease simulation with several variable parameters and data visualizations. The main goal of this program with to allow each agent to run on its own thread as an introduction to concurrency.

Design:

Each agent runs on its own thread and only can communicate to their immediate neighbor. Agents use a blocking queue to avoid liveness problems and communicate with the main thread to determine their neighbors. Each of the data visualizations were run on the main thread and each agent communicated its state to the main thread. Agents move at constant velocity and change their heading by a random amount. If an agent went out of bounds its heading would be flipped by 180 degrees.

Parameters

The disease parameter was designed to be variable and were able to be adjusted while the simulation was running. This was achieved by giving the agent class private static variables that could be changed via public setters. Additionally an import/ export system was added to allow for parameters to be saved an loaded at any time.

What I learned:

I learned how to effectively use many concurrent threads in a program while avoid common liveness problems such as deadlock. I also learned about how JavaFX works, and that JFX components can only be modified from the main JFX thread. This gave me greater insight into how large programs work and leverage modern day multi-core/ multi-threaded CPUs.

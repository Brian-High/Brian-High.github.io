Scrabble Game and Solver

School Project

Java  ·   JavaFX
TRIE data structure  ·  Dictionaries  ·   Modular Design  ·  Algorithm Efficiency

Objective:

Create two programs, a Scrabble game and a Scrabble board solver, that are object-oriented and highly modular. The Scrabble game had a GUI that used the board solver program to play against the user.

Solver:

A dictionary file was read in and stored in a TRIE (preffix tree). Using the dictionary, all the possible words that could be made by the tray were found and stored. All the valid words were placed in various positions on the board. Then each board state was searched for invalid words and scored.

Output

Performance:

The Algorithm's performance was acceptable for this assignment, but far from ideal. The worst case scenario was a 21 x 21 board with 2 wild cards (tiles that can be any letter). These wildcards really blew up the Algorithm because they increased the of possible words by 2 orders of magnitude. Even in this case, the longest time the algorithm ran for was about 30s, while smaller configurations ran almost instantly.

What I Learned:

I learned a lot about planning ahead and leveraging data structures to optimize an algorithm. I was able to improve upon a first attempt, brute force style algorithm by embracing a "fail-fast" approach. The goal was to check the word as it was being placed onto the board to see if it was valid before checking all of the other parts of the board. Also the only part of the board that needed to be checked was the area around where the new word was placed. Another area that was improved upon greatly was using the TRIE to determine what letter a wild card could be rather than check all 26 possible letters.

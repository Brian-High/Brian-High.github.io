Gone Fishin'

Professional Application - Ideum

Unity  ·   C#/.NET
Content Management System  ·  Google Sheets/Drive API  ·   Shaders  ·  Path Finding

Objective:

Create and educational and interactive program that runs on a touchscreen and wall mount display. This application was a remake of an existing exhibit at the Fort Worth Zoo from the 80s. The goal was to make this exhibit simpler and convey Texas's unique rules and regulations of in a concise manner. Note: at the time of my leaving Ideum, the final assets where still being designed so the visuals here reflect placeholder assets

Design:

The original exhibit had a physical pond with fish cut outs that visitors could catch with real fishing poles. This exhibit would often break and did not convey the information in a fun and concise manner. The new program used a virtual pond on a touch screen where users touch to "cast" a bobber and the fish would path find to the bobber. After a fish has been "caught" a card is displayed on a second monitor displaying important information.

Content Management System (CMS)

This program was designed use to dynamic data so the client can to change the content of the app after it is deployed. Using the Ideum custom dev stack, I was able to leverage the Google Sheets and Google Drive API to pull in text and image data from the cloud. The Ideum Sheets API converts the a google sheet's rows into a list of delimited strings where the text data and media URLs can be extracted. Then the media URLs are then passed to the Drive API where the file is downloaded to appdata and information like the file name and extension can be accessed. This text data and media file information is then Serialized into a JSON file using the NewtonSoft JSON serializer. After this process has been done once, all the data is stored locally and only needs to be pulled when content is updated.

Shaders:

The water effect in pond was achieve using multiple shaders in conjunction with a render texture. There was a 3D mesh in a plane that was deformed by a vertex shader to give the effect of waves. A separate component of the shader would respond to touch points and create ripples that deformed the mesh too. Finally, a separate pixel shader would create the water shifting and glare effect by modifying the color with some math. These shaders were then applied render texture that is an overlay on the pond.
